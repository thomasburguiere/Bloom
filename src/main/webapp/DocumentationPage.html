Mapping to DarwinCore tool

The mapping tool help to convert your file to a standard : the DarwinCore.

What's the DarwinCore ?

The Darwin Core is body of standards. It includes a glossary of terms (in other contexts these might be called properties, elements, fields, columns, attributes, or concepts) intended to facilitate the sharing of information about biological diversity by providing reference definitions, examples, and commentaries. The Darwin Core is primarily based on taxa, their occurrence in nature as documented by observations, specimens, samples, and related information.

source : TDWG.org

The DarwinCore standard contains many terms, explained here. To use our workflow, your data have to be in this DarwinCore format. To be easier for you, your data are read (csv format necessary at least) and all columns name are displayed. If your column name is recognized by the DarwinCore standard, so this one are automatically mapped. If not, you can choose the correct DarwinCore term corresponding.

Figure 1 : example for mapping DarwinCore tool

On the left side, there are your columns, and on the right side, DarwinCore terms. In this example, the "establishmentMeans" term is recognized by the tool. The "isoCode" isn't recognized, so you can choose the equivalent term thanks to the list : "countryCode".

Masks validation tool

The mask validation tool valid or not each occurencce are correctly included in raster file.

What's a raster file ?

In its simplest form, a raster consists of a matrix of cells (or pixels) organized into rows and columns (or a grid) where each cell contains a value representing information, such as temperature. Rasters are digital aerial photographs, imagery from satellites, digital pictures, or even scanned maps. The dimension of the cells can be as large or as small as needed to represent the surface conveyed by the raster dataset and the features within the surface. For more information, go to ArcGIS.

To explain the mask validation tool, I'll take an example (figure 1).

Figure 2 : example for mask validation tool

For this example (figure 1), an occurence is georeferenced and marked by red cross. The geolocalisation is correct : the red cross is inside a land (green). There are three differents raster from the smallest (left grid) to the largest (right grid). Consequently, for the first raster, occurence is included in a valid cell, but not for the other.

Where is the problem ?

Tools for raster treatment like (?????) use informations contained in cells. Sometimes, cells have informations for land cells and not for water cells. For the last, these programs don't suport these occurences. Often, a filtering is necessary before.

Our mask validation tool do this checking to set appart all wrong occurences. To use this validation, you have to insert one or more raster files. VERIFIER FORMAT PRIS EN CHARGE. You can found raster files here : WorldClim

Taxonomic Reconciliation

Thanks to a reference, scientific names can be checked

What's the taxonomic reconciliation ?

The taxonomic reconciliation is a tool that, given some text which is a name or label for something, and optionally some additional details, returns a ranked list of potential entities matching the criteria. The candidate text does not have to match each entity's official name perfectly, and that's the whole point of reconciliation--to get from ambiguous text name to precisely identified entities.

source : OpenRefine-Reconciliation-Service
